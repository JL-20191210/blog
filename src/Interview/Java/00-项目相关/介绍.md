# 项目介绍1.0

好的面试官。接下来我来介绍一下项目。
		我参与的项目是某大容量数据分布式管理系统。整个系统的主要功能是通过各种侦查手段获取侦查信息，并将侦查到的信息作为需求提交给卫星公司，由卫星公司根据要求拍摄卫星图像，进行数据处理后返回图像等数据信息给部队方，最终实现监视敌方目标的功能。	我主要负责其中的 侦查需求统筹系统 和 侦查需求辅助处理系统。
		先讲 侦查需求统筹系统，它是三级节点架构：从低到高依次是哨位级、处科级、基地级。我在其中负责核心功能开发，主要包括需求录入、需求管理、需求提报、需求消解、状态同步。
		统筹系统采用 Spring Boot + Spring Cloud Alibaba 微服务架构，服务拆分为需求管理、数据处理、状态同步、报表导出模块，利用 Nacos 实现服务注册，负载均衡和配置热更新，从而实现服务的高可用。跨服务通信采用OpenFeign同步调用和RabbitMQ异步通信，跨节点通信采用 RabbitMQ 异步消息传递，还采用了Redis 缓存高频查询结果，保障系统的响应速度。在数据处理方面，我使用 Java 8 Stream 流式分组 + 异步线程池并行计算，显著提高需求消解效率。
		在跨服务操作中，为了解决分布式事务一致性问题，我们引入了 Seata。在需求消解和状态同步模块中，涉及多个服务的数据更新，我通过 Seata 的 AT 模式保证了各服务间数据的一致性，避免了跨服务操作因部分失败导致的数据不一致问题。
		通过优化索引与 SQL 查询结构，并设计 Redis 缓存刷新策略，将查询性能从秒级提升至毫秒级，大幅提升系统响应速度与用户体验。
		再讲 侦查需求辅助处理系统，它作为顶级节点，承担全国所有需求的集中处理。系统同样采用微服务架构。功能包括需求接收、消解、提报、状态同步和报表导出。
		报表导出模块采用 EasyExcel 高性能生成 Excel 文件，结合 SQL 索引优化及redis缓存策略。将报表生成从分钟级提升到秒级。同时系统全流程自动化处理，流量削峰，显著降低人工操作，提高业务效率和可靠性。



## 项目介绍2.0

好的面试官。我介绍一下我参与的这个项目。

​		我参与的这个项目是某大容量数据分布式管理系统。是部队的一个重点项目，主要功能是通过各种侦查手段获取的侦查信息，将获取到的侦查信息作为一条需求，通过这个系统传递给卫星公司，由卫星公司根据需求拍摄卫星影像，进行数据处理后将影像元数据信息回传给系统。由系统将图像元数据信息分发给需求发起方，需求发起方再根据影像信息和侦查信息生成情报，最终实现监视敌方目标的功能。这个系统解决了部队获取情报信息时效性差，步骤繁琐的一个痛点。以前获取情报需要人工打电话给卫星公司，由卫星公司编制拍摄计划，然后通过ftp的形式传递影像数据，中间会有一些刻盘导入导出的工作，非常繁琐，并且卫星公司的计划执行情况不透明。这个项目实现了从获取侦查信息到接收影像信息自动化执行，而且可以让用户实时监控任务的执行状态。

​		整个系统是分为01和02两个大板块的，其中02包括IO和存储两个大系统，IO负责对接不同的卫星公司同时接收来自的01的需求信息。存储负责接收卫星公司回传的影像元数据信息并进行分发。01系统包括各种侦查手段系统，侦查需求统筹系统和侦查需求辅助处理系统。各种侦查手段系统获取侦查信息后传递给统筹系统，由统筹系统进行数据处理后向上传递给辅助处理系统，辅助处理系统作为01的出口和IO系统进行数据的交互。

​		我在这个项目中负责的就是侦查需求统筹系统和侦查需求辅助处理系统。首先说一下统筹系统，统筹系统是分为三级节点的，是一个树形结构，从上到下分别是局基地级节点，科处级节点和哨位级节点。一个战区就是一个局基地级节点。我在其中负责核心功能开发，主要包括需求录入、需求管理、需求提报、需求消解、状态同步。

​		统筹系统采用 Spring Boot + Spring Cloud Alibaba 微服务架构，服务拆成了需求管理、数据处理、状态同步、报表导出几个模块，用docker部署了多个副本并注册到nacos上，用 Nacos 实现服务注册发现，负载均衡和配置集中管理，保证服务的高可用。跨服务调用的时候是用的网关和OpenFeign进行同步调用，异步调用用的是消息队列RabbitMQ，不同节点之间也是用的消息队列rabbitmq。对于需要高频查询的信息采用了redis进行缓存提高查询效率。为了解决分布式事务一致性问题，我们还使用了 Seata进行事务的管理。

​		再讲 侦查需求辅助处理系统，它作为顶级节点，承担全国所有需求的集中处理。系统同样采用微服务架构。功能包括需求接收、消解、提报、状态同步和报表导出。
​		报表导出模块采用 EasyExcel 高性能生成 Excel 文件，结合 SQL 索引优化及redis缓存策略。将报表生成从分钟级提升到秒级。同时系统全流程自动化处理，流量削峰，显著降低人工操作，提高业务效率和可靠性。
